# Auto-compression-jpg 插件 PRD

## 产品概述

**插件名称**: Auto-compression-jpg  
**插件类型**: Eagle 后台服务插件 (Background Service Plugin)  
**核心功能**: 自动检测今天添加的 JPG 文件，并用本地"图压"应用打开进行自动压缩

---

## 功能需求

### 1. 自动检测与压缩

- **检测条件**: 
  - 文件格式：JPG/JPEG
  - 添加时间：今天添加到 Eagle 的文件（基于 `importedAt` 或 `modifiedAt`）
  
- **执行方式**:
  - 使用本地"图压"应用打开符合条件的文件
  - macOS 命令：`open -a "图压" "文件路径"`
  
- **去重机制**:
  - 每个文件只打开一次
  - 使用 `processed.json` 记录已处理的文件 ID
  - 每日自动重置记录（跨日清空）

### 2. 定时任务

- **执行频率**: 默认每 1 分钟检查一次
- **可配置范围**: 1-60 分钟
- **后台运行**: 插件启动后自动在后台执行，无需用户手动触发

### 3. 手动批量压缩

- **时间筛选选项**: 最近 7/14/30/60/90 天，或所有图片
- **智能去重**: 自动跳过已有"已图压压缩"标签的图片
- **确认机制**: 显示找到的文件数量，用户确认后执行
- **进度反馈**: 
  - 每处理 10 个文件输出进度日志
  - 处理完成后显示成功通知

### 4. 自动标签

- **标签名称**: "已图压压缩"
- **添加时机**: 打开图压应用后立即添加（不等待压缩完成）
- **应用范围**: 
  - 自动后台任务处理的文件
  - 手动批量压缩处理的文件
- **去重机制**: 检查标签是否已存在，避免重复添加

### 5. 配置界面

**界面元素**:

1. **启用/禁用开关**
   - 复选框控制自动压缩功能的开关
   - 默认：关闭

2. **检查间隔设置**
   - 数字输入框
   - 范围：1-60 分钟
   - 默认：1 分钟

3. **图压应用路径**
   - 文本输入框
   - 默认值：`/Applications/图压.app`
   - 用户可自定义应用路径

4. **手动批量压缩按钮组** (新增)
   - 6 个时间筛选按钮（7/14/30/60/90 天，所有）
   - 2 列网格布局
   - 仅处理未标记的图片

5. **保存设置按钮**
   - 保存配置到 `config.json`
   - 保存后立即应用新配置（重启定时器）

6. **状态显示**
   - 上次运行时间
   - 今日已处理文件数量

### 4. 数据持久化

**配置文件 (config.json)**:
```json
{
  "enabled": false,
  "intervalMinutes": 1,
  "appPath": "/Applications/图压.app"
}
```

**已处理记录 (processed.json)**:
```json
{
  "lastDate": "2026-02-03",
  "processedIds": ["id1", "id2", "id3"]
}
```

**存储位置**: 
- macOS: `~/Library/Application Support/Eagle/plugins/{PLUGIN_ID}/`
- 使用 `eagle.app.userDataPath` 或 `process.env.HOME` 作为基础路径

## 技术实现

### 核心技术栈

- **Eagle Plugin API**: 
  - `eagle.item.get()` - 获取 JPG 文件列表
  - `eagle.onPluginCreate()` - 插件初始化
  - `eagle.onPluginShow()` - 显示设置界面
  
- **Node.js 原生模块**:
  - `child_process.exec` - 执行系统命令打开图压应用
  - `fs` - 读写配置和已处理记录
  - `path` - 路径处理

- **定时器**: `setInterval()` 实现定时任务

### 关键流程

```
插件启动
  ↓
加载配置 (config.json)
  ↓
启动定时器 (延迟 1 秒初始化)
  ↓
每 N 分钟执行：
  1. 获取所有 JPG/JPEG 文件
  2. 筛选今天添加的文件
  3. 排除已处理的文件
  4. 逐个用图压打开（间隔 500ms）
  5. 记录已处理的文件 ID
  ↓
保存处理记录 (processed.json)
```

### 错误处理

- **API 初始化失败**: 使用 `try-catch` + 环境变量 fallback
- **延迟初始化**: `setTimeout(..., 1000)` 确保 Eagle API 就绪
- **路径转义**: 正确处理含空格和中文的文件路径
- **缓存机制**: `cachedStorageDir` 避免重复计算存储路径

## 使用场景

### 典型工作流

1. **用户添加大量截图到 Eagle**
   - 用户每天从网页/设计工具截取大量 JPG 图片
   - 手动逐个压缩太繁琐

2. **插件自动介入**
   - 后台每 1 分钟检测新增的 JPG 文件
   - 自动用"图压"应用打开并压缩
   - 图压压缩后自动保存，替换原文件

3. **无感知压缩**
   - 用户无需手动操作
   - 每个文件只压缩一次，避免重复处理
   - 释放磁盘空间，优化存储

## 注意事项

1. **系统兼容性**: 
   - 当前版本仅支持 macOS
   - 需要本地安装"图压"应用

2. **Eagle 版本要求**:
   - Eagle 4.0 build18+ (支持 `importedAt` 字段)
   - 较旧版本使用 `modifiedAt` 作为备选

3. **性能考虑**:
   - 每次打开图压间隔 500ms，避免系统负载过高
   - 大量文件时可能需要较长处理时间

4. **路径处理**:
   - 正确转义含空格和特殊字符的路径
   - 支持中文文件名和路径

5. **Eagle API 注意事项**:
   - ⚠️ `eagle.item.get()` 使用 `fields` 参数可能导致返回空对象
   - **推荐做法**: 不指定 `fields` 参数，获取完整的 item 对象
   - 使用 `importedAt` 或 `modifiedAt` 字段判断添加时间
